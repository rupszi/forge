Forge — UI mockups

Five clickable HTML designs of the Forge desktop experience. Open any one to see it full-size; the layout adapts to your viewport.
01

Main chat — in-session

Three-column dashboard: project + sessions sidebar, central chat with planner output and live sprint sub-cards, agents panel on the right. Top bar: context-window meter, free/Ollama tier badge, model. Composer with attach menu, slash commands, mode picker, transcript verbosity, Create-PR.
02

New session — empty state

Welcome screen with hero composer, suggestion chips (fix bug, new feature, add tests, refactor, …), four example prompt cards seeded for the user's stack, and the slash-command palette demo expanded.
03

Merge gate — diff review

File-tree on the left grouped by sprint, syntax-highlighted diff in the middle, evaluator's per-criterion verdict + multi-perspective review (security/correctness/performance) on the right. "Approve & merge" + "Reject selected" actions. Knowledge-extraction prompt at bottom.
04

First-run wizard

Detected stack at top (next, supabase, stripe, github), already-configured connectors auto-checked, suggested connectors based on stack signals. Anti-corruption contract callout at the bottom. Step 2 of 4 progress.
05

Knowledge base browser

Three-pane KB browser: project nav, faceted filters (category × topic × source × confidence), and the result list with confidence bars, usage counters, source provenance, and inline edit / delete / inspect actions. Stats row up top.
These mockups live at mockups/ in the repo. Open mockups/index.html in any browser; or hand each individual file straight to a designer / contributor for review. They use the same design tokens as the existing Next.js dashboard at ui/, so anything we accept in mockups translates directly.
